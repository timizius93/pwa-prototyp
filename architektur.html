Ausgangslage
Warum eine PWA?
Die bisherige Magazin-App ist tot. Die PWA ersetzt sie — ohne App Store, ohne doppeltes Layout, ohne externe Abhängigkeiten.
Problem
Button Publish (InDesign-Plugin) — der einzige Weg zur bisherigen nativen App — ist ausgefallen. SSL-Zertifikat abgelaufen, kein Support, kein Fallback. Doppeltes Layout (iPad + Smartphone), App Store Abhängigkeit, kein Offline, kein Web.
Lösung
Progressive Web App auf einer Subdomain (mag.enduro-mtb.com). Responsive statt doppeltes Layout, installierbar ohne App Store, offline-fähig, werbefinanziert via Standard Web-Ads. Erscheint 3–4× pro Jahr als kuratierte Ausgabe — ergänzt das tägliche News-Portal auf enduro-mtb.com.

Workflow
Vom Inhalt zur fertigen Ausgabe
Komponenten werden einmal gebaut. Danach ist jede Ausgabe nur noch Content — kein Layout, kein App Store Review, kein Warten.
01
Komponenten-Design (einmalig)
Art Director gestaltet die Komponenten-Templates: Cover, Article-Hero, Bike-Test, Fullbleed-Photo etc. Jede Komponente wird einmal designed — und gilt für alle zukünftigen Ausgaben.
Figma
02
Komponenten-Implementierung (einmalig)
Developer baut die Komponenten als wiederverwendbaren Code. Parallel werden die Sanity-Schemas definiert: welche Felder hat ein Bike-Test? (Marke, Preis, Federweg, Score-Werte…)
Next.js Sanity
03
Content-Erstellung (pro Ausgabe)
Redaktion tippt Texte, lädt Bilder hoch, füllt Scores und Specs aus. Art Director wählt und ordnet Komponenten — welche Seiten kommen in welcher Reihenfolge?
Sanity CMS
04
Live-Vorschau & Review
Sanity zeigt die echte PWA neben dem Editor in Echtzeit. Korrekturen direkt im CMS, kein Datei-Austausch, kein PDF-Feedback.
Sanity Presentation
05
Veröffentlichung
"Publish" in Sanity triggert automatischen Build. Keine App Store Reviews, keine Wartezeiten. Die neue Ausgabe ist in Minuten live.
